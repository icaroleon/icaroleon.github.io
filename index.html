Sobre Mim

Após ter atuado como advogado criminalista em escritórios tanto em Minas Gerais quanto no Rio de Janeiro, decidi seguir minha paixão pela tecnologia e me tornar desenvolvedor em 2022. Desde o Ensino Médio era ficcionado com as possibilidades existentes na área de tecnologia, inclusive, já programando como hobby antes mesmo de começar a faculdade de Direito.

No entanto, sabia que esse movimento traria inúmeros desafios, como a incerteza de iniciar em um ramo completamente diverso e a variedade de materiais disponíveis na internet. Diferentemente da minha experiência no Direito, não há um caminho formal e fixo na área de tecnologia, o que gerou confusão no início.

Para contornar esses desafios, realizei o bootcamp de desenvolvimento web da Le Wagon, momento no qual, além dos diversos ensinamentos pessoais adquiridos, tive contato com diversas linguagens, conceitos, funcionalidades e desafios da área, com destaque para Ruby on Rails. Além disso, em 2023, iniciei uma nova graduação em Análise e Desenvolvimento de Sistemas para complementar e aprofundar ainda mais meus estudos.

Atualmente, busco uma oportunidade profissional para colocar em prática meus ensinamentos e ter contato com o ambiente de criação de produtos digitais

Movies API

Tecnologias empregadas:

Ruby on Rails; PostgreSQL; RSpec; Postman.

Principais desafios:
  • Primeira API em Ruby on Rails;
  • Definir os testes que seriam utilizados para a verificação pelo RSpec;
Descrição:

Desafio realizado para uma oportunidade backend. O teste consistiu em criar uma API de serviço do catálogo de filmes com dois endpoints: um que faça a leitura de um arquivo CSV para registrá-los no banco de dados. E um segundo que liste todos os filmes cadastrados em formato JSON. Para tanto, deveria se utilizar Ruby on Rails, seguir o padrão de API Restful, além de que a response deveria seguir um padrão pré estabelecido ordenado pela data de lançamento do filme, bem como não deve possuir duplicidade de registros. Veja o repósitório do código para acessar as instruções de acesso à API.

Utilizando o framework Ruby on Rails, além de autenticações para usuários, APIs tais quais Mapbox (para o serviço de geolocalização), Cloudinary (para o upload de fotos), Websocket (para mensagens privadas entre usuários), além de nested comments (para viabilizar respostas aos posts principais de tal evento, respostas à tais respostas, e assim por diante), o Let's Go é um aplicativo web que se assemelha à uma rede social, possibilitando ao usuário anunciar eventos (públicos ou privados), confirmar presença, conversar com seus amigos, e comentar nos posts dos eventos divulgados (Veja o código).

Ruby on Rails, Javascript, HTML/CSS, Bootstrap, PostgreSQL, Git, Cloudinary.
